# Milestone v4.0: Extension Core

**Status:** SHIPPED 2026-02-04
**Phases:** 24-29
**Total Plans:** 18

## Overview

This roadmap delivered the Chrome Extension MVP with COPILOTO mode — a manual row-advancement workflow where users fill forms from Excel data and confirm success before proceeding. The journey started with extension infrastructure (WXT + Manifest V3), established backend support (extension auth + row status), implemented auth flow and popup controls, built the DOM manipulation layer, and culminated in the integrated fill cycle that orchestrates all components.

## Milestone Summary

**Key Decisions:**
- WXT 0.20.13 over CRXJS (built-in storage/messaging abstractions)
- Service Worker state via chrome.storage (30-second termination risk)
- Native property setters for React/Vue form compatibility
- Jose library for extension JWT (modern async API, no CommonJS issues)
- In-memory lockout map for brute force protection
- Discriminated union message types for type-safe messaging
- Module state for current mapping selection (vs. preferences)

**Issues Resolved:**
- React peer dependency hoisting (React 19 at workspace root)
- Content script sendMessage undefined result causing crash
- Fill progress clearing during active fill
- Row navigation not resetting fill status

**Issues Deferred:**
- Redis lockout storage for multi-instance deployments
- JWT key rotation mechanism
- TypeScript errors in messaging modules (non-blocking)

**Technical Debt Incurred:**
- In-memory lockout state lost on API restart (acceptable for dev)
- 6-digit codes limited to 1M combinations (mitigated by lockout + expiry)
