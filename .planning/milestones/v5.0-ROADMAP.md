# Milestone v5.0: Mapping Builder

**Status:** SHIPPED 2026-02-05
**Phases:** 30-34
**Total Plans:** 20

## Overview

v5.0 enables users to create mappings visually by clicking form fields in the extension, edit them in the dashboard, and see meaningful row identifiers while filling. Five phases: backend foundation for project/batch CRUD and batch identifiers, dashboard management UI, dashboard mapping editor, extension capture mode, and extension identifier integration.

**Plans**: 3 plans

## Milestone Summary

**Key Decisions:**
- Cascade soft-delete without transaction (idempotent, retryable)
- Config replaces list view for batch settings (modal-free, simpler navigation)
- dnd-kit in extension (consistent DnD behavior across apps)
- Storage-based step sync in capture (popup cannot receive messages when closed)
- Background as message router (popup cannot directly message content scripts)
- Module-level captureMode instance (persists across messages)
- Store identifier keys at batch selection (fetch once, use for all rows)
- createPortal for DragOverlay (prevents clipping by overflow containers)
- Navigation arrows hidden for single-row batches (reduces UI clutter)

**Issues Resolved:**
- Batch inline edit blocker (CRUD-03 form submission fix)
- Missing toast feedback on batch settings save
- Card button visual inconsistencies (cosmetic polish)
- Mapping editor card layout clipping issue
- Form state sync with external data on React 19
- Drag-and-drop portal overlay for proper z-index
- Row navigation arrows missing from RowIndicator

**Issues Deferred:**
- ID-04 human verification: manual HTTP test for row identifier values in response format
- Cascade delete human verification: create project with batch and rows, DELETE project, verify all soft-deleted

**Technical Debt Incurred:**
- Pre-existing TypeScript errors in extension src/messaging/handlers.ts (unrelated to v5.0)
- Pre-existing ESLint configuration issue: missing eslint.config.js in extension
