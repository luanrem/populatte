Create `src/content/capture/selector-gen.ts`:
- Import `getCssSelector` from css-selector-generator
- Define `SelectorResult` interface with `primary` and optional `fallbacks` (each has `type: 'css'` and `value: string`)
- Export `generateSelector(element: HTMLElement): SelectorResult` function:
  - Generate primary selector with options:
    - selectors: ['id', 'class', 'tag', 'attribute']
    - blacklist: [/^js-/, /^ng-/, /^react-/, /^vue-/, /^_\w+/, /\w+-[a-f0-9]{6,}/] (framework classes)
    - combineWithinSelector: true
    - combineBetweenSelectors: true
    - maxCombinations: 10
  - Generate fallback selector with nth-child only:
    - selectors: ['tag', 'nthchild']
    - includeTag: true
  - Return { primary: { type: 'css', value }, fallbacks: [{ type: 'css', value: fallbackSelector }] }
- Export `detectAction(element: HTMLElement): 'fill' | 'click'` function:
  - Return 'fill' for INPUT, TEXTAREA, SELECT
  - Return 'click' for everything else (BUTTON, A, etc.)

Per RESEARCH.md: css-selector-generator handles edge cases (shadow DOM, dynamic IDs, framework classes, uniqueness testing).

Create `src/content/capture/highlighter.ts` implementing element highlighting:

**Class: ElementHighlighter**

Private properties:
- `overlay: HTMLDivElement | null` - Blue outline overlay
- `tooltip: HTMLDivElement | null` - Element type + name hint tooltip
- `isActive: boolean`
- `abortController: AbortController | null` - For cleanup via signal
- `onCapture: ((element: HTMLElement) => void) | null` - Callback for captured element

Methods:

`activate(onCapture: (element: HTMLElement) => void): void`
- Set isActive = true
- Create abortController = new AbortController()
- Inject styles (using style element, not inline - CSP safe)
- Create overlay and tooltip elements, append to body
- Add mouseover listener on document (with signal from abortController):
  - If target is interactive element (INPUT, SELECT, TEXTAREA, BUTTON, A), show overlay and tooltip
  - Otherwise hide overlay
- Add click listener on document (with signal):
  - event.preventDefault(), event.stopPropagation()
  - If target is interactive element, call onCapture(target)
  - Show brief pulse animation (150ms per CONTEXT.md)

`deactivate(): void`
- Set isActive = false
- abortController.abort() - removes all listeners automatically
- Remove overlay and tooltip from DOM
- Remove injected style element

`private showOverlay(element: HTMLElement): void`
- Get element.getBoundingClientRect()
- Position overlay with fixed positioning at element bounds
- Blue border (2px solid #0066ff)
- z-index: 2147483647 (maximum safe)
- pointer-events: none
- opacity transition 150ms

`private showTooltip(element: HTMLElement): void`
- Create tooltip content: element type + name/id hint
  - Example: "Input: email" or "Button: submit"
  - Format: `${element.tagName.toLowerCase()}: ${element.name || element.id || 'unnamed'}`
- Position tooltip above element (top - 28px)
- Handle collision with viewport top (show below instead)
- Same high z-index, pointer-events: none

`private isInteractiveElement(el: HTMLElement): boolean`
- Return true for: INPUT, SELECT, TEXTAREA, BUTTON, A
- Per CONTEXT.md: "All clickable elements highlightable (inputs, selects, buttons, links)"

`private injectStyles(): void`
- Create style element with CSS for overlay and tooltip:
  - .populatte-capture-overlay { position: fixed; border: 2px solid #0066ff; ... }
  - .populatte-capture-tooltip { position: fixed; background: #0066ff; color: white; ... }
  - .populatte-capture-pulse { animation: populatte-pulse 150ms ease-out; }
  - @keyframes populatte-pulse { from { box-shadow: 0 0 0 0 rgba(0,102,255,0.4); } to { ... } }

**File: badge-tracker.ts**

Class: BadgeTracker
- Tracks captured elements and displays numbered badges at top-right corner
- Uses IntersectionObserver (not polling) per RESEARCH.md for performance

Private properties:
- `badges: Map<HTMLElement, HTMLDivElement>` - Element to badge mapping
- `observer: IntersectionObserver`
- `stepCounter: number` - Auto-increment for badge numbers

Constructor:
- Create IntersectionObserver with threshold [0, 0.1, 0.9, 1]
- Callback: when entry.isIntersecting changes, update badge visibility and position

Methods:

`addBadge(element: HTMLElement): number`
- Increment stepCounter
- Create badge div with number (blue circle, white text, 24x24px)
- Store in badges Map
- Position badge at element top-right corner
- observer.observe(element)
- Append badge to document.body
- Return stepCounter (for step number)

`removeBadge(element: HTMLElement): void`
- Get badge from Map
- observer.unobserve(element)
- Remove badge from DOM
- Delete from Map

`getBadgeNumber(element: HTMLElement): number | null`
- Return badge number if element is tracked, null otherwise

`updateBadgePosition(element: HTMLElement): void`
- Get badge from Map
- Update position based on getBoundingClientRect()
- Top-right corner: top = rect.top - 12, left = rect.right - 12

`cleanup(): void`
- observer.disconnect()
- Remove all badges from DOM
- Clear Map
- Reset stepCounter

**File: capture-mode.ts**

Class: CaptureMode (main coordinator)

Private properties:
- `highlighter: ElementHighlighter | null`
- `badgeTracker: BadgeTracker | null`
- `isActive: boolean`
- `capturedElements: Map<HTMLElement, CapturedStep>` - Track what was captured

Interface CapturedStep:
- `stepNumber: number`
- `selector: SelectorResult`
- `action: 'fill' | 'click'`
- `elementType: string`
- `elementName: string`

Methods:

`activate(): void`
- If already active, return
- Set isActive = true
- Create highlighter and badgeTracker
- highlighter.activate(this.handleElementCaptured)

`deactivate(): void`
- highlighter?.deactivate()
- badgeTracker?.cleanup()
- Clear capturedElements
- Set instances to null, isActive = false

`getCapturedSteps(): CapturedStep[]`
- Return Array.from(capturedElements.values()) sorted by stepNumber

`removeStep(stepNumber: number): void`
- Find element with matching stepNumber
- badgeTracker.removeBadge(element)
- Delete from capturedElements

`isElementCaptured(element: HTMLElement): boolean`
- Return capturedElements.has(element)

`private handleElementCaptured = (element: HTMLElement): void`
- If element already captured, return (per CONTEXT.md: show prompt - but popup handles this)
- Generate selector using generateSelector(element)
- Detect action using detectAction(element)
- Add badge using badgeTracker.addBadge(element)
- Create CapturedStep and store in capturedElements
- Send message to popup: chrome.runtime.sendMessage({ type: 'ELEMENT_CAPTURED', payload })
