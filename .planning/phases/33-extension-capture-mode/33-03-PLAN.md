Create `entrypoints/popup/components/capture/StepList.tsx`:

**Types:**

```typescript
export interface CaptureStep {
  id: string; // Temporary ID for list management
  stepNumber: number;
  action: 'fill' | 'click' | 'wait';
  selector?: {
    type: 'css';
    value: string;
  };
  fallbacks?: Array<{ type: 'css'; value: string }>;
  elementType?: string;
  elementName?: string;
  sourceFieldKey?: string;
  fixedValue?: string;
  clearBefore?: boolean;
  pressEnter?: boolean;
  waitMs?: number;
  optional?: boolean;
}

interface StepListProps {
  steps: CaptureStep[];
  onReorder: (steps: CaptureStep[]) => void;
  onEdit: (stepId: string) => void;
  onDelete: (stepId: string) => void;
  onHighlight: (stepNumber: number) => void;
}
```

**Component implementation:**

- Import from @dnd-kit/core: DndContext, closestCenter, DragEndEvent, PointerSensor, useSensor, useSensors
- Import from @dnd-kit/sortable: arrayMove, SortableContext, verticalListSortingStrategy, useSortable
- Import from @dnd-kit/utilities: CSS

- Use useSensors with PointerSensor (delay 150ms to distinguish from click)
- Handle drag end: arrayMove steps, update stepNumber for each, call onReorder
- Empty state: "Click elements on the page to capture steps" (per CONTEXT.md)
- Each step item shows:
  - Drag handle (grip icon)
  - Step number badge
  - Action type icon (Pencil for fill, MousePointer for click, Clock for wait)
  - Source: column name, fixed value, or "(click only)" / wait duration
  - Truncated selector (max 30 chars)
  - Edit button (pencil icon) - calls onEdit
  - Delete button (trash icon) - calls onDelete (instant, no confirmation per CONTEXT.md)
- Single-click on row calls onHighlight (scrolls element into view)

**SortableStepItem subcomponent:**

- Use useSortable hook
- Apply transform and transition styles
- Drag handle has {...attributes} {...listeners}
- Proper cursor and visual feedback during drag

Style with Tailwind classes matching popup design (compact, rounded-lg borders).

**File: StepConfig.tsx**

Step configuration panel that replaces step list view when editing/adding.

Props:
```typescript
interface StepConfigProps {
  step: CaptureStep | null; // null for new wait step
  columns: string[]; // Available batch columns
  onSave: (step: CaptureStep) => void;
  onCancel: () => void;
  isNew: boolean; // true for new step, false for edit
}
```

Implementation:
- Action radio group: fill, click, wait (per CONTEXT.md)
  - Auto-detect from element type for new captures
  - fill handles inputs, selects, checkboxes
  - click for buttons, links
  - wait for manual delay steps
- Source section (only for fill action):
  - Toggle between "Column" and "Fixed Value" modes
  - Column mode: searchable dropdown with columns array
  - Fixed value mode: text input
- Wait section (only for wait action):
  - Quick preset buttons: 500ms, 1s, 2s (per CONTEXT.md)
  - Custom input for other values
- Options section (checkboxes):
  - optional: "Optional (skip if not found)"
  - clearBefore: "Clear field before filling" (fill only)
  - pressEnter: "Press Enter after filling" (fill only)
- Selector display (read-only, per CONTEXT.md)
- "Highlight Target" button - sends message to highlight element
- Action buttons:
  - "Add Step" (isNew) or "Save Changes" (edit) - calls onSave
  - "Cancel" - calls onCancel

Validation on submit only (not inline per CONTEXT.md):
- fill action requires sourceFieldKey or fixedValue
- wait action requires waitMs > 0

**File: CapturePanel.tsx**

Main capture mode container.

Props:
```typescript
interface CapturePanelProps {
  targetUrl: string; // Current page URL
  columns: string[]; // From selected batch
  onSave: (name: string, steps: CaptureStep[]) => void;
  onCancel: () => void;
}
```

State:
- `mappingName: string` - Name input
- `steps: CaptureStep[]` - Captured steps
- `editingStep: CaptureStep | null` - Currently editing step
- `showConfig: boolean` - Toggle between list and config view
- `addingWait: boolean` - Adding new wait step

Implementation:
- Header with "Capture Mode" title and step count badge (per CONTEXT.md: "5 steps")
- Mapping name input field at top
- Conditional view:
  - If showConfig: show StepConfig
  - Else: show StepList
- StepList handlers:
  - onReorder: update steps with new order
  - onEdit: set editingStep, showConfig = true
  - onDelete: remove step, send CAPTURE_REMOVE_STEP message
  - onHighlight: send CAPTURE_HIGHLIGHT message
- "Add Wait" button at bottom of list (per CONTEXT.md)
- Footer with:
  - "Cancel" button (confirm if steps.length > 0)
  - "Save" button (disabled until name + 1 step per CONTEXT.md)

Handle ELEMENT_CAPTURED message:
- Create new CaptureStep from payload
- Add to steps array
- Optionally auto-open config for new step
