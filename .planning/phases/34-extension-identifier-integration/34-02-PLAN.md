<objective>
Add standalone Prev/Next navigation arrows to RowIndicator so users can browse rows independently of the fill workflow.

Purpose: The UAT found that users cannot navigate rows because the Next button in FillControls is gated by fillStatus (only enabled after a fill completes). The background script already handles ROW_PREV and ROW_NEXT messages, but the popup has no UI for free row navigation. This plan adds arrow buttons to RowIndicator that work whenever rowTotal > 1, completely independent of fill state.

Output: RowIndicator with clickable Prev/Next arrows, App.tsx with handlePrev function and navigation props wired through.
</objective>

**RowIndicator.tsx changes:**

1. Import `ChevronLeft` and `ChevronRight` from `lucide-react`.

2. Add two new props to `RowIndicatorProps`:
   - `onPrev: () => void`
   - `onNext: () => void`

3. Restructure the row counter line ("Row X of Y") into a flex row with three elements:
   - Left: Prev button (`ChevronLeft` icon, `w-5 h-5`) - a small round button
   - Center: "Row {rowIndex + 1} of {rowTotal}" text (same `text-xs text-gray-500` as current)
   - Right: Next button (`ChevronRight` icon, `w-5 h-5`) - a small round button

4. Button styling:
   - Base: `p-0.5 rounded-full hover:bg-gray-200 transition-colors text-gray-500 hover:text-gray-700`
   - Disabled: `disabled:opacity-30 disabled:cursor-not-allowed disabled:hover:bg-transparent disabled:hover:text-gray-500`
   - Keep buttons compact so they don't bloat the indicator

5. Disable logic:
   - Prev button: `disabled={rowIndex <= 0}`
   - Next button: `disabled={rowIndex >= rowTotal - 1}`
   - Both buttons hidden entirely when `rowTotal <= 1` (use conditional rendering, not just disabled)

6. Click handlers:
   - Prev button `onClick`: call `onPrev()`, also call `e.stopPropagation()` to prevent the copy-to-clipboard from firing
   - Next button `onClick`: call `onNext()`, also call `e.stopPropagation()` to prevent copy-to-clipboard

7. The existing click-to-copy behavior on the outer div MUST be preserved. The arrow buttons sit inside the outer clickable div, so `e.stopPropagation()` is essential to avoid copy-on-navigate.

8. The "No rows" empty state (`rowTotal === 0`) remains unchanged - no arrows shown.

**App.tsx changes:**

1. Add `handlePrev` async function (mirror of existing `handleNext` at line 147-155):
   ```
   async function handlePrev() {
     setFillError(null);
     try {
       await sendToBackground<VoidResponse>({ type: 'ROW_PREV' });
     } catch (err) {
       console.error('[Popup] Failed to go to previous row:', err);
     }
   }
   ```

2. Update the `<RowIndicator>` JSX (around line 379-386) to pass the new props:
   - `onPrev={handlePrev}`
   - `onNext={handleNext}` (reuse the existing handleNext function)

NOTE: The existing `handleNext` in App.tsx sends `ROW_NEXT` to the background script. The background script already resets fillStatus to idle on ROW_NEXT (line 386). This is correct behavior for navigation arrows -- the fill workflow Next button in FillControls remains separate and still gated by fillStatus. Both use the same handleNext function which is fine because the background handler is the same.
