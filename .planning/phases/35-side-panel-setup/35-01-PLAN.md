<task type="auto">
  <name>Task 2: Update WXT config and background for sidepanel activation</name>
  <files>
    apps/extension/wxt.config.ts
    apps/extension/entrypoints/background.ts
  </files>
  <action>
    1. **Update `wxt.config.ts`**: Add `sidePanel` to the permissions array in manifest config. The current permissions are `['storage', 'activeTab', 'scripting']`. Add `'sidePanel'` to this array. WXT auto-generates `side_panel.default_path` from the `entrypoints/sidepanel/` directory, so no manual `side_panel` manifest key is needed.

    2. **Add `setPanelBehavior` to background.ts**: At the TOP of the `defineBackground()` callback (after the storage initialization block), add:
       ```typescript
       // Enable Side Panel on icon click (replaces popup)
       browser.sidePanel.setPanelBehavior({ openPanelOnActionClick: true }).catch((err) => {
         console.error('[Populatte] Failed to set panel behavior:', err);
       });
       ```
       This makes clicking the extension icon toggle the Side Panel open/closed (per locked decision: standard Chrome toggle behavior). The `browser.sidePanel` API is available because WXT polyfills Chrome APIs via the `browser` namespace.

    3. **Build the extension** to verify:
       - Run `npm run build --filter=@populatte/extension` from the monorepo root.
       - Check the generated manifest in `.output/chrome-mv3/manifest.json` to verify:
         - `side_panel.default_path` points to the sidepanel HTML
         - `permissions` includes `"sidePanel"`
         - There is NO `action.default_popup` key (popup removed)

    NOTE on WXT type definitions: If TypeScript complains that `browser.sidePanel` doesn't exist, the WXT/webextension-polyfill types may not include the sidePanel API. In that case, use `(chrome as any).sidePanel.setPanelBehavior(...)` or add a type declaration:
    ```typescript
    declare namespace chrome {
      namespace sidePanel {
        function setPanelBehavior(behavior: { openPanelOnActionClick: boolean }): Promise<void>;
      }
    }
    ```
    Place the declaration at the top of background.ts or in a `src/types/chrome.d.ts` file.

    IMPORTANT: Do not modify any other background.ts logic (message handlers, state management, broadcast). Plan 02 handles the port-based communication refactor.
  </action>

    - The built manifest contains `"sidePanel"` in permissions array
    - The built manifest does NOT contain `"default_popup"` under `"action"`
    - TypeScript compilation passes (`npm run type-check --filter=@populatte/extension` if available, otherwise build success suffices)
