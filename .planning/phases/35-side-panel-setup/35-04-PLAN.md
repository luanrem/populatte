<objective>
Convert ProjectSelector and BatchSelector from sendToBackground (runtime.sendMessage) to sendViaPort (port-based messaging), closing the last 2 gaps blocking Phase 35 completion.

Purpose: Both components send messages via runtime.sendMessage which routes to the content-script handler (runtime.onMessage). But GET_PROJECTS and GET_BATCHES are only handled in the port listener in background.ts. This causes "Unknown message type" errors after login, making the extension unusable beyond authentication.

Output: Both selectors use port-based messaging, matching the pattern established in Plan 35-03 for CodeInputForm.
</objective>
