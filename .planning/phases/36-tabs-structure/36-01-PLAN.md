Create `TabBar.tsx` with the following specs:

**Props interface:**
```typescript
interface TabBarProps {
  activeTab: 'preencher' | 'captura';
  onTabChange: (tab: 'preencher' | 'captura') => void;
  captureActive: boolean; // Controls whether Captura tab is enabled + shows pulsing dot
}
```

**Tab bar design (per user decisions — these are LOCKED):**
- Two tabs: "Preencher" and "Captura", equal width (w-1/2 each, flex)
- Icon + text label for each tab:
  - Preencher: `FileText` icon from lucide-react (or `ClipboardList` — Claude's discretion on exact icon)
  - Captura: `Target` or `Crosshair` icon from lucide-react (Claude's discretion)
  - Small icon (w-4 h-4) before text label
- Shadcn-style aesthetic: Use Tailwind classes matching shadcn/ui tab patterns:
  - Container: full width, flex, border-b border-border (or gray-200)
  - Active tab: text-foreground, border-b-2 border-primary (or a visible indicator)
  - Inactive tab: text-muted-foreground
  - Tab buttons: flex items-center justify-center gap-1.5, py-2, text-sm font-medium
- NO transitions or animations on tab switch (instant swap per user decision)
- Default cursor on hover for all tabs (NOT pointer for disabled — user decision)

**Disabled state (Captura when captureActive is false):**
- Grayed out: opacity-50 or text-muted-foreground with reduced opacity (Claude's discretion on exact level)
- Click does NOT switch tabs — instead shows tooltip
- Tooltip text: "Inicie a captura primeiro" (user decision — exact text)
- Use `title` attribute for tooltip (simplest approach, consistent with extension's existing patterns)
- Preencher tab is ALWAYS accessible, never disabled

**Pulsing dot badge (when captureActive is true):**
- Blue pulsing dot (bg-blue-500) to the RIGHT of "Captura" text
- Small dot: w-2 h-2 rounded-full
- Pulsing animation: use Tailwind `animate-pulse` class
- Only shown when `captureActive` is true
- When captureActive is true, Captura tab is clickable (not disabled)

**Implementation notes:**
- This is a controlled component — activeTab and onTabChange come from parent
- Do NOT use shadcn's actual Tabs component (it's not installed in the extension). Hand-craft the tab bar using Tailwind to LOOK like shadcn tabs.
- The tab separator between tab bar and content is a border-b on the container (Claude's discretion on exact styling)

Export from `index.ts`: Add `export { TabBar } from './TabBar';`
