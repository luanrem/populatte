Wait — re-reading the decision more carefully: "Default tab on fresh open: always Preencher" and "On reopen: default to Preencher, BUT if capture mode is active, open on Captura". This means the tab memory from preferences is NOT used to restore tab on reopen. The only override is capture mode. So `lastActiveTab` in preferences serves no purpose for reopen behavior.

However, the requirement TAB-04 says "remembers the last active tab across reopens". Let me reconcile: the user said "default to Preencher" on reopen but also said "remembers last active tab". The reconciliation is: remember the tab, but override to Preencher unless capture is active. So the tab memory exists but only matters if we later need it. For now, the behavior is:
- Always open on Preencher
- UNLESS capture mode is active -> open on Captura

Still persist `lastActiveTab` on tab change (for future use and TAB-04 compliance), but initialization always defaults to Preencher with capture mode override.

This ensures the tab preference is always current. Even though reopen defaults to Preencher, storing it keeps TAB-04 satisfied and supports future extensions.
