  <action>
In `handleEnterCaptureMode()` (around line 281), move `setActiveTab('captura')` to execute BEFORE the async operations. The current order is:

```
fetchBatchDetail (network call) → sendViaPort (port message) → chrome.storage.session.set → setCaptureMode(true) → setActiveTab('captura')
```

Change to:

```
setActiveTab('captura') → setCaptureMode(true) → fetchBatchDetail → sendViaPort → chrome.storage.session.set
```

Specifically:
1. Move `setActiveTab('captura')` to immediately after the guard clause (`if (!portRef.current || ...)`) and before the `try` block.
2. Move `setCaptureMode(true)` right after `setActiveTab('captura')`, also before the `try` block.
3. In the `catch` block, add rollback: `setCaptureMode(false)` and `setActiveTab('preencher')` to revert if async operations fail.
4. Remove the now-redundant `setCaptureMode(true)` and `setActiveTab('captura')` from inside the try block (lines 305-307).
5. Keep `setBatchColumns(columns)` inside the try block since it depends on the fetched data.

This gives an optimistic UI update: the tab switches instantly, then the async work happens in the background. If it fails, we roll back.
