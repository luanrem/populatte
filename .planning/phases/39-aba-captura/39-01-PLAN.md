<objective>
Fix capture mode exit flows so they return the user to the Preencher tab.

Purpose: The capture mode UI (CapturePanel, StepList, StepConfig) and all supporting infrastructure (tab switching, session storage persistence, content script integration, "Criar Mapping" button wiring) already exist and work correctly. However, when capture mode ends — either via "Comecar a Preencher" after saving or via "Cancelar" — the handlers set `captureMode = false` without switching `activeTab` back to `'preencher'`. This leaves the user stranded on a disabled Captura tab. This plan adds the missing `setActiveTab('preencher')` calls.

Output: Working App.tsx with complete capture mode lifecycle (enter -> capture -> exit returns to Preencher).
</objective>
