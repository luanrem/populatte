<objective>
Implement compact mode for the Side Panel -- a toggle that collapses the full Preencher/Captura UI into a minimal icon grid for rapid visual inspection and click-to-highlight workflows.

Purpose: Power users filling many rows can reduce visual noise to a compact icon strip, quickly inspect elements on the page, then expand back to the full UI for filling. This is the final phase of v5.1.

Output: CompactIconGrid component, storage persistence for compact mode, toggle button in header, keyboard shortcut, and CSS transition wiring in App.tsx.
</objective>

    **1. Create CompactIconGrid.tsx:**

    **Props interface:**
    ```typescript
    interface CompactIconGridProps {
      steps: MappingStep[];
      validation: Map<string, boolean>;
      onStepClick: (step: MappingStep) => Promise<void>;
    }
    ```

    **CompactIconGrid component:**
    - Render a scrollable grid container: `<div className="grid grid-cols-3 gap-2 p-2 overflow-y-auto flex-1">`
    - Map over `steps` rendering a `CompactStepIcon` for each
    - Pass `step`, `isInvalid` (from `validation.get(step.id) === false`), and a click handler

    **CompactStepIcon component (internal to the file):**
    - Props: `step: MappingStep`, `isInvalid: boolean`, `onStepClick: (step: MappingStep) => Promise<void>`
    - Local state: `const [clickResult, setClickResult] = useState<'success' | 'failed' | null>(null);`

    - **Action icon selection** (same logic as PreencherStepList's SortableStepItem):
      - `fill` -> `Pencil`
      - `click` -> `MousePointer`
      - `wait` -> `Clock`
      - Import from `lucide-react`

    - **Tooltip text:** Build from `[step.action, step.selector?.value ?? '', step.sourceFieldKey ?? step.fixedValue ?? ''].filter(Boolean).join(' | ')`
      - For wait steps: use `${step.waitMs ?? 0}ms` instead of selector

    - **Click handler:**
      - Call `onStepClick(step)` which returns a Promise (the existing `handleStepHighlight` in App.tsx)
      - After the promise resolves, determine result: if the call didn't throw, show `success` badge; if it threw, show `failed` badge
      - Actually, `handleStepHighlight` returns void and handles errors internally. Instead, use a simpler approach: wrap in try/catch, show 'success' on resolve, 'failed' on catch
      - Set `clickResult` state, then clear it after 1500ms via `setTimeout`
      - Use a ref to track the timeout and clear it on unmount or re-click

    - **Render structure:**
    ```tsx
    <button
      type="button"
      onClick={handleClick}
      className="relative flex items-center justify-center w-full aspect-square bg-gray-50 hover:bg-gray-100 rounded-lg border border-gray-200 group/tooltip transition-colors"
    >
      <ActionIcon className="w-6 h-6 text-gray-600" />

      {/* Step number badge - top-right, notification style */}
      <span className="absolute top-0.5 right-0.5 w-4 h-4 flex items-center justify-center text-[10px] font-bold bg-blue-100 text-blue-700 rounded-full">
        {step.stepOrder}
      </span>

      {/* Invalid selector warning - top-left, always visible (per user decision) */}
      {isInvalid && (
        <span className="absolute top-0.5 left-0.5">
          <AlertTriangle className="w-3.5 h-3.5 text-amber-500" />
        </span>
      )}

      {/* Group-hover tooltip (Phase 37 pattern - instant, no delay) */}
      <div className="absolute bottom-full left-1/2 -translate-x-1/2 mb-1 px-2 py-1 bg-gray-900 text-white text-xs rounded whitespace-nowrap opacity-0 invisible group-hover/tooltip:opacity-100 group-hover/tooltip:visible transition-opacity pointer-events-none z-50">
        {tooltipText}
      </div>

      {/* Temporary click result badge - bottom-right corner */}
      {clickResult && (
        <span className={`absolute bottom-0.5 right-0.5 w-3 h-3 rounded-full ${
          clickResult === 'success' ? 'bg-green-500' : 'bg-red-500'
        }`} />
      )}
    </button>
    ```
